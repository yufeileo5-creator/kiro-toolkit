# 多模态生成工具包 (Multimodal Toolkit)

> 迁移自 skills/_on-demand/minimax-multimodal-toolkit/SKILL.md。通过 #multimodal-toolkit 手动引用。
> 使用 MiniMax API 生成语音、音乐、视频和图片。

## 前置条件
- 设置 MINIMAX_API_KEY 和 MINIMAX_API_HOST 环境变量
- 安装 ffmpeg 和 jq

## 能力矩阵

| 能力 | 说明 |
|------|------|
| TTS | 文本转语音，多声音多情感 |
| 语音克隆 | 从音频样本克隆声音（10s-5min） |
| 语音设计 | 从文字描述创建自定义声音 |
| 音乐生成 | 生成带歌词的歌曲或纯音乐 |
| 图片生成 | 文生图、图生图（角色参考） |
| 视频生成 | 文生视频、图生视频、主体参考、模板 |
| 长视频 | 多场景链式视频 + 交叉淡入淡出 |
| 媒体工具 | 格式转换、拼接、裁剪、提取 |

## TTS 关键规则
- 单声音直接用 tts 命令，不要拆分 segments
- 多角色/旁白+对话才用 segments.json + generate 命令
- 旁白和对话必须拆分为独立 segment

## 音乐关键规则
- 为视频/语音配 BGM 时默认用纯音乐模式（--instrumental），不要问用户
- 用户明确要求"创作音乐"时才询问是否要歌词

## 图片关键规则
- 根据用户意图推断宽高比（风景 16:9、人像 2:3、手机壁纸 9:16 等）
- 默认 1 张，用户说"几张"时生成 3 张

## 视频关键规则
- 默认 10s + 768P，除非用户明确要求长视频
- 1080P 只支持 6s
- 生成前必须优化 prompt：主体 + 场景 + 运动 + 镜头 + 氛围
- 图生视频时 prompt 只描述运动和变化，不重复图片内容

## 长视频规则
- 第 1 段：文生视频（完整场景描述）
- 第 2+ 段：图生视频（上一段末帧为首帧，prompt 描述变化）
- 各段用 0.5s 交叉淡入淡出拼接
