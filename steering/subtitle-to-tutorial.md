# 字幕转教程 (Subtitle to Tutorial)

> 迁移自 skills/_on-demand/subtitle-to-tutorial。通过 #subtitle-to-tutorial 手动引用。
> 将 TXT/SRT 格式的授课逐字稿转换为高质量 Markdown 教程。

## 核心原则

1. 唯一知识源，纯粹 JS：基准源自《The Complete JavaScript Course》，严禁出现 Java 内容
2. 教学风格：深度模仿 Udemy 讲课风格，极度注重底层原理，慢节奏逐行解析
3. 具体案例先行 (Concrete Before Abstract)：讲解任何抽象概念前，必须先提供"灾难代码沙盒"展示痛点
4. 零容忍教学搭桥：检测到"概念空降"时，即使讲师说"以后再讲"，也必须用 :::bridge 原地扫盲
5. 忠于原词：忠实于传入文本的核心逻辑
6. 中文首选：专业术语首次出现保留英文并标注中文
7. 拒绝伪代码：所有示例必须完整可运行
8. 可视化与纯代码UI化：必须包含 Mermaid 图解和生活类比
9. 无损全谱系转写：严禁粗暴合并讲师的多步代码推演，保留试错、报错、修补全过程
10. 格式兼容铁律：fenced code block 必须顶格书写，严禁 4 空格缩进

## 避坑指南

1. 超前预支幻觉：AI 在基础语法中强行插入高阶 API → 用 :::pro-tip 明确标注"未来预告"
2. 干瘪口诀背诵：直接甩口诀无实战铺垫 → 必须先配 :::runnable "灾难现场沙盒"
3. "以后会讲"的敷衍病：讲师说"先不用懂" AI 就真不教 → 用 :::bridge 强行就地启蒙
4. 核心概念跳级病：主标题核心概念不铺垫 → 必须先上"不用它的惨烈后果"沙盒
5. Markdown 嵌套崩溃：:::fill-blank 等短交互块忘记反引号 → 严格用反引号隔离

## 转换工作流（6 阶段）

1. 解析 SRT 文件提取内容
2. 判断课程类型（普通课 or 实战挑战）
3. 选择模板（普通课用 lesson_template，实战课用 challenge_template）
4. 内容填充与沙盒生成
5. 自我质量检查（对照质检清单）
6. 输出 Markdown 制成品

## 自定义组件语法

| 组件 | 用途 |
|------|------|
| :::bridge | 零容忍前置搭桥，新名词原地扫盲 |
| :::recall | 记忆闪回，复习前置知识 |
| :::mission | 战前简报，明确学习目标和获得感 |
| :::mental | 认知重塑，用生活类比解释新概念 |
| :::compare | 代码对比（Bad vs Pro） |
| :::sandboxing | 破坏性实验，让读者动手搞崩代码 |
| :::tracer | V8 级执行追踪，模拟单步调试 |
| :::pro-tip | 顶级开发者视角，工程最佳实践 |
| :::pitfall | 灾难现场与避坑指南 |
| :::loot | 战利品清点，能力雷达总结 |
| :::runnable | 可运行代码沙盒 |
| :::fill-blank | 代码填空交互 |
| :::quiz | 测验卡片 |

## 普通课模板结构

1. 标题 + 来源 + 章节
2. 知识脉络（前置知识 + 后续扩展）
3. :::bridge 零容忍搭桥（如需要）
4. :::recall 记忆闪回（如需要）
5. :::mission 战前简报
6. 核心知识点：
   - 灾难痛点体验（:::runnable 灾难沙盒）
   - 概念与解药（:::mental 认知重塑 + 生活类比）
   - 高保真演进实战（:::runnable + :::compare）
   - 视觉透视图（Mermaid 或纯代码 UI）
   - 破坏性实验（:::sandboxing）
   - 原理剖析与执行追踪（:::tracer）
   - 顶级开发者视角（:::pro-tip）
   - 避坑指南（:::pitfall）
7. 记忆口诀
8. 词汇速查表
9. 代码实战与真实场景
10. :::loot 战利品与能力雷达

## 实战课模板结构

1. 标题 + 来源 + 章节
2. 业务动机
3. 场景漫游沙盒（完整可执行代码，用户无需手写）
4. 导师视角拆解

## 质检清关单（交付前必须逐条审查）

### 绝对红线

1. :::tracer 滥用防御：只用于逐行演示 JS 内存变动和执行栈，非时序性理论分析降级为标准块引用
2. Mermaid 图审查：
   - 禁止用 CSS div 假冒图表，必须用正规 Mermaid
   - 所有节点文本必须包裹在英文双引号中（如 A["文本"]）
   - 节点文本中禁止夹带 HTML 标签
3. 痛点先行准则：新概念首次登场必须先有灾难沙盒，禁止干巴巴抛抽象名词
4. 代码块排版：fenced code block 必须 0 缩进顶格

### 质量优化

1. AI 废话剔除：删掉"总之"、"首先"、"综上所述"等机械八股文
2. 零容忍伪代码：禁止 // ... 在这里写代码，必须给完整上下文
3. 类比审查：深奥底层逻辑必须有生活类比

## 文件命名

- 格式：{序号}_{标题}.md
- 示例：003 Destructuring Arrays.en.srt → 003_Destructuring_Arrays.md
