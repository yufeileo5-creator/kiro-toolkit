# 全局代理规则 (Agent Rules)

> 完整规则详见 `#agent-rules-full`。

## 七条始终生效铁律

1. **强制中文** — 所有对话、注释、文档使用中文，代码标识符保留英文
2. **禁止占位符** — 严禁 `// ... existing code`，所有修改必须完整可运行
3. **需求先审后答** — 新需求先做信息补全审查，严禁立刻给方案或写代码
4. **分层约束** — 严禁反向依赖，新第三方库须先申请
5. **验证后交付** — 说"完成"前必须运行验证，证据优先于断言
6. **Steering 优先** — 进入项目先读项目级 steering 规则
7. **环境自举** — 新项目无基础配置时，引用 `#project-templates` 初始化

## 核心行为约束

- 文件修改前用中文简述意图、风险、预期
- 复杂需求禁止立刻写码，先产出实施计划待批准
- 遇到错误：四阶段诊断（根因调查→模式分析→假设验证→实施修复），严禁盲目重试
- 连续 3 次修复失败：停下来质疑架构，与用户讨论
- 反摆烂：任务失败 2+ 次或用户不满时，穷尽一切方案

## 黄金准则路由

按场景引用 `#golden-rules` 中对应章节：
- 修改 import/新增模块 → 架构准则
- 每次代码修改 → 代码质量准则
- 功能完成后 → 文档准则（配合 `#doc-sync-check`）
- 遇到已知错误 → 错误自修复提示
